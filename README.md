# 이벤트 페이지 프로젝트

신규 서비스 오픈 기념 이벤트 페이지입니다. 룰렛 게임, 카운트다운 타이머, 응모 폼 등의 기능을 제공합니다.

## 실행 방법

### 1. 의존성 설치

```bash
npm install
```

### 2. 개발 서버 실행

```bash
npm run dev
```

개발 서버가 실행되면 브라우저에서 `http://localhost:5173` (또는 터미널에 표시된 주소)로 접속할 수 있습니다.

### 3. 프로덕션 빌드

```bash
npm run build
```

빌드된 파일은 `dist` 폴더에 생성됩니다.

### 4. 빌드 결과물 미리보기

```bash
npm run preview
```

## 사용한 라이브러리 및 선택 이유

### 핵심 프레임워크

#### Vue 3 (^3.5.13)
- **선택 이유**: 
  - 컴포넌트 기반 아키텍처로 재사용 가능한 UI 구성 요소 개발에 적합

  - 반응성 시스템이 강력하여 실시간 업데이트가 필요한 카운트다운 타이머 구현에 최적
  - TypeScript와의 통합이 우수하여 타입 안정성 확보

#### TypeScript (~5.9.3)
- **선택 이유**:
  - 정적 타입 검사를 통해 런타임 에러 사전 방지
  - 인터페이스를 통한 API 응답 데이터 구조 명확화
  - IDE 자동완성 및 리팩토링 지원으로 개발 생산성 향상

### 빌드 도구

#### Vite (^7.2.4)
- **선택 이유**:
  - 빠른 개발 서버 시작 시간과 HMR(Hot Module Replacement) 지원
  - ESBuild 기반의 빠른 번들링 속도
  - 최적화된 프로덕션 빌드 생성
  - Vue 3와의 공식 통합으로 안정적인 개발 환경 제공

### 스타일링

#### Tailwind CSS (^4.1.18)

  - 유틸리티 클래스 기반으로 빠른 스타일링 가능

  - 커스텀 설정을 통한 디자인 시스템 일관성 유지
  - 번들 크기 최적화 (사용하지 않는 스타일 자동 제거)

### 애니메이션

#### GSAP (^3.14.2)
- **선택 이유**:
  - 고성능 애니메이션 라이브러리로 부드러운 사용자 경험 제공
  - Timeline API를 통한 복잡한 애니메이션 시퀀스 제어
  - 다양한 이징 함수로 자연스러운 애니메이션 효과 구현
  - 컴포넌트 등장 애니메이션(fade-in, slide-up) 구현에 적합

### 게임 라이브러리

- **선택 이유**:

## 구현 시 고민한 점

### 1. 룰렛 게임 구현

#### 문제점
- Winwheel.js가 기본적으로 TweenMax/TweenLite를 사용하는데, 프로젝트에서는 GSAP를 사용 중
- 룰렛 회전 후 정확한 세그먼트에 멈추도록 하는 각도 계산의 복잡성

#### 해결 방법
- GSAP를 TweenMax/TweenLite 인터페이스로 래핑하여 호환성 레이어 구현
- 포인터 각도(-90도)와 세그먼트 중심 각도를 고려한 정확한 목표 회전 각도 계산
- 여러 바퀴 회전(9회전) 후 목표 세그먼트에 정확히 멈추도록 `stopAngle` 설정
- 애니메이션 완료 콜백과 폴백 타이머를 함께 사용하여 안정성 확보

### 2. 반응형 디자인

#### 문제점
- 다양한 화면 크기에서 일관된 사용자 경험 제공 필요
- 룰렛 캔버스 크기를 화면 크기에 맞게 동적으로 조정 필요

#### 해결 방법
- Tailwind CSS의 반응형 유틸리티 클래스 활용 (`sm:`, `md:`, `lg:` 등)
- 룰렛 캔버스 크기를 `computed` 속성으로 계산하여 화면 크기에 따라 자동 조정
- 카운트다운 타이머의 폰트 크기와 패딩을 반응형으로 설정
- `flex-shrink-0` 클래스를 사용하여 작은 화면에서도 레이아웃 깨짐 방지

### 3. API 통신 및 에러 처리

#### 문제점
- 네트워크 오류나 API 실패 시 사용자 경험 저하
- 중복 응모 방지를 위한 데이터 검증 필요

#### 해결 방법
- 로딩 상태와 에러 상태를 명확히 구분하여 UI에 반영
- 에러 발생 시 사용자 친화적인 메시지와 재시도 버튼 제공
- 이메일과 전화번호 정규화(소문자 변환, 하이픈 제거)를 통한 중복 검사 정확도 향상
- `try-catch` 블록을 통한 예외 처리 및 사용자에게 명확한 피드백 제공

### 4. 애니메이션 성능

#### 문제점
- 여러 컴포넌트가 순차적으로 등장하는 애니메이션 구현
- 애니메이션이 사용자 경험을 방해하지 않도록 자연스러운 타이밍 필요

#### 해결 방법
- GSAP Timeline을 사용하여 여러 애니메이션을 하나의 시퀀스로 관리
- `-=0.4` 오프셋을 사용하여 애니메이션 간 겹침을 통한 자연스러운 흐름 구현
- `power3.out` 이징 함수로 부드럽고 자연스러운 감속 효과 적용
- 초기 상태를 `gsap.set()`으로 설정하여 FOUC(Flash of Unstyled Content) 방지

### 5. 카운트다운 타이머 정확도

#### 문제점
- API에서 받은 종료 시간을 기준으로 정확한 카운트다운 표시
- 1초마다 업데이트되면서도 성능에 영향이 없어야 함

#### 해결 방법
- `setInterval`을 사용하여 1초마다 남은 시간 재계산
- `computed` 속성을 활용하여 일, 시, 분, 초를 효율적으로 계산
- `padStart`를 사용하여 항상 2자리 숫자로 표시 (예: "09" 대신 "9" 방지)
- 컴포넌트 언마운트 시 인터벌 정리로 메모리 누수 방지

### 6. 폼 검증 및 중복 체크

#### 문제점
- 사용자가 이미 응모한 경우 중복 응모 방지
- 이메일과 전화번호 형식의 다양성 (하이픈 포함/미포함)

#### 해결 방법
- 응모 전에 전체 엔트리 목록을 가져와 이메일/전화번호 중복 검사
- 데이터 정규화(소문자 변환, 하이픈 제거)를 통한 검사 정확도 향상
- 중복 발견 시 명확한 에러 메시지 제공
- 클라이언트 측 검증과 서버 측 검증 모두 구현

### 7. 상태 관리

#### 문제점
- 여러 컴포넌트 간 데이터 공유 (이벤트 정보, 응모 상태 등)
- 비동기 데이터 로딩 상태 관리

#### 해결 방법
- Vue 3의 `ref`와 `reactive`를 사용한 간단한 상태 관리
- Props와 Emits를 통한 부모-자식 컴포넌트 간 데이터 전달
- API 서비스를 별도 모듈로 분리하여 관심사 분리
- TypeScript 인터페이스를 통한 데이터 구조 명확화

## 주요 기능

- 🎰 **룰렛 게임**: 6개 세그먼트로 구성된 룰렛을 돌려 보상을 받을 수 있습니다
- ⏰ **카운트다운 타이머**: 이벤트 종료까지 남은 시간을 실시간으로 표시합니다
- 🎁 **보상 목록**: 이벤트에서 제공하는 보상들을 시각적으로 확인할 수 있습니다
- 📝 **응모 폼**: 이름, 전화번호, 이메일을 입력하여 이벤트에 응모할 수 있습니다
- 🔒 **중복 방지**: 이메일과 전화번호를 기반으로 중복 응모를 방지합니다
- ✨ **애니메이션**: GSAP를 활용한 부드러운 페이지 전환 및 컴포넌트 등장 애니메이션
